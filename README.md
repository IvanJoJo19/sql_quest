# SQL Quest: SQL-тренажер с ИИ-помощником

SQL Quest - это учебный веб-тренажер для изучения SQL через практические задания. Пользователь выбирает задание, пишет SQL-запрос, запускает его в PostgreSQL-песочнице, видит результат в таблице и получает автоматическую проверку решения.

В проект также добавлен endpoint ИИ-помощника: он дает подсказки по заданию и запросу пользователя. Если API-ключ LLM не задан, подсказки работают в локальном мок-режиме.

## Что реализовано в MVP

- Список учебных SQL-заданий.
- SQL-редактор в браузере.
- Выполнение SQL-запросов через backend.
- Реальная PostgreSQL-база `sql_quest`.
- Учебные таблицы `students`, `courses`, `enrollments`.
- Задания по темам `SELECT`, `WHERE`, `ORDER BY`, `JOIN`, `GROUP BY`.
- Табличный вывод результата.
- Автоматическая проверка ответа.
- Прогресс прохождения.
- Endpoint `/api/hint` для LLM-подсказок.
- Мок-режим ИИ-помощника без API-ключа.
- Запуск на Windows через `start_backend.bat`.

## Ценность проекта

Новичкам сложно изучать SQL только по теории. SQL Quest дает безопасную среду, где можно сразу писать запросы, видеть результат, ошибаться, получать подсказки и постепенно проходить задания.

Пользователь получает:

- практику вместо сухой теории;
- мгновенный результат SQL-запроса;
- автоматическую проверку;
- подсказки от ИИ-помощника;
- понятный прогресс обучения.

## Как запустить

### Запуск через exe

Локальная Electron-сборка находится здесь:

```text
dist/win-unpacked/SQL Quest.exe
```

Также на рабочем столе создан ярлык:

```text
SQL Quest EXE
```

При запуске exe приложение:

- запускает Python backend;
- подключается к локальному PostgreSQL;
- открывает SQL Quest в отдельном desktop-окне.

Важно: текущий exe рассчитан на компьютер разработчика, где уже установлены Python и PostgreSQL.

### Полная версия с backend и PostgreSQL

На компьютере разработчика:

1. Запустить файл:

```text
start_backend.bat
```

2. Дождаться открытия браузера или открыть вручную:

```text
http://127.0.0.1:8000
```

3. Не закрывать окно терминала, пока приложение используется.

Для остановки запустить:

```text
stop_backend.bat
```

### Быстрый просмотр интерфейса

Если backend и PostgreSQL не установлены, можно открыть:

```text
index.html
```

Так можно посмотреть интерфейс, но SQL-запросы через PostgreSQL и подсказки работать не будут.

## Что нужно для запуска на другом компьютере

Для полноценного запуска нужны:

- Python;
- PostgreSQL;

## PostgreSQL

Backend:

- запускает локальный PostgreSQL;
- создает базу `sql_quest`, если ее еще нет;
- загружает учебные данные из `data/seed.sql`;
- выполняет запросы через endpoint `/api/query`;
- разрешает только безопасные `SELECT`-запросы;
- добавляет ограничение `LIMIT 100`, если пользователь не указал лимит.

Учебные таблицы:

- `students` - студенты, город, возраст, уровень.
- `courses` - курсы, сложность, длительность.
- `enrollments` - записи студентов на курсы, прогресс и оценка.

## LLM API

В проекте реализован endpoint:

```text
/api/hint
```

Он принимает текущее задание и SQL-запрос пользователя.

Сейчас ИИ-помощник работает в мок-режиме, потому что API-ключ не задан.
## Стек

Текущий MVP:

- Frontend: HTML, CSS, JavaScript.
- Backend: Python `http.server`.
- База данных: PostgreSQL.
- Данные: SQL seed-файл `data/seed.sql`.
- LLM: OpenAI-compatible API через HTTP, с fallback мок-режим.

Планируемое развитие:

- Backend: FastAPI.
- Frontend: React + TypeScript.
- База данных: PostgreSQL с отдельными учебными схемами.
- Контейнеризация: Docker Compose.
- LLM: полноценные подсказки, объяснение ошибок и генерация новых заданий.

## Безопасность SQL-песочницы

добавлены базовые ограничения:

- разрешены только запросы, начинающиеся с `SELECT`;
- запрещены `DROP`, `DELETE`, `UPDATE`, `INSERT`, `ALTER`, `TRUNCATE`, `CREATE`, `COPY`, `GRANT`, `REVOKE`;
- запрещены несколько SQL-команд в одном запросе;
- запрещены SQL-комментарии;
- автоматически добавляется `LIMIT 100`, если пользователь не указал лимит.

В будущем можно усилить изоляцию:

- отдельный пользователь PostgreSQL только с правами чтения;
- отдельная учебная схема;
- таймаут выполнения запросов;
- пересоздание учебной базы перед сессией.

## Источники данных

Для MVP используются сгенерированные учебные данные.

В дальнейшем можно добавить:

- демо-датасеты PostgreSQL;
- открытые учебные SQL-наборы с GitHub;
- собственные датасеты преподавателя.

## Структура проекта

- `index.html` - основной экран приложения.
- `styles.css` - стили интерфейса.
- `app.js` - логика frontend, задания, отправка SQL на backend.
- `backend.py` - Python backend, PostgreSQL и LLM endpoint.
- `start_backend.bat` - запуск приложения на Windows.
- `stop_backend.bat` - остановка backend-процессов.
- `launch_sql_quest.ps1` - вспомогательный launcher.
- `data/seed.sql` - схема и учебные данные PostgreSQL.
- `.env.example` - пример настроек LLM API.
